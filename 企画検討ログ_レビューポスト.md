# レビューポスト（ReviewPost）— 企画検討ログ

> 作成日: 2026-04-03
> ステータス: 企画検討中
> 技術: Google Stitch + Gemini API

---

## コンセプト

食後にQRコードを読み取り、簡単な質問に答えるだけで、AIがおしゃれな感想カードを自動生成。来店客が自分のSNSに自然と投稿したくなる仕組みで、飲食店の口コミ・集客を支援するアプリ。

---

## 3人チーム討論の経緯

Google Stitchを使った飲食店向けアプリを、3つの視点で並列検討した。

### 検討した視点
| メンバー | 担当 |
|----------|------|
| メンバー1 | ビジネス価値・収益性・クライアントニーズ |
| メンバー2 | 技術・セキュリティ設計・実現性 |
| メンバー3 | UX・顧客体験・エンゲージメント |

### 全体で出たアイデア（9案）
1. AIメニューナビ（MenuTalk）— 全員推薦
2. AI口コミ返信アシスタント（ReplyChef）— 2人推薦
3. **AI感想カード＆口コミ生成（レビューポスト）— Stitchのデザイン力を最も活かせる**
4. AIフードフォト加工（DishUp）
5. AI接客トレーニング（ServeCoach/TrainMate）
6. 日替わりPOP自動生成（POPスタジオ）
7. AI食材ロス削減（FoodSaver AI）
8. 料理のストーリーテリング（テーブルストーリー）
9. グループ注文コラボ（オーダーパル）

### レビューポストを選んだ理由
- **Stitchのデザインセンスを最大限に活かせる**（カードの美しさ＝アプリの価値）
- 個人情報ゼロで実現可能（セキュリティ方針に完全合致）
- 導入ハードル最低（QRコードを置くだけ）
- SNS拡散の仕組みが内蔵されている

## 利用フロー

### 店舗側：導入（初回のみ・5分）

1. レビューポストのサイトにアクセス
2. 店舗名・ジャンル（居酒屋/カフェ等）を入力
3. カードデザインのテーマを選択（和モダン/ナチュラル/ポップ/シック等）
4. 店のロゴ画像をアップロード（任意）
5. 専用QRコードが自動生成される
6. QRコードを印刷してテーブルやレジ横に設置

### 来店客：食後の体験（30秒〜1分）

1. QRコードを読み取る（レシートやテーブルPOPから）
2. 3つの質問に答える
   - Q1. 今日のベスト料理は？（タップで選択）
   - Q2. 今日の気分は？（😍最高 😊満足 😋また来る）
   - Q3. ひとこと（任意・自由入力）
3. AIが感想カードを生成（店のテーマに合ったデザイン）
4. シェア（任意）— 保存/X/Instagram/LINE

### 店舗側：効果確認

- ダッシュボードで匿名集計データを確認
  - カード生成数
  - 人気メニューTOP3
  - 気分の分布

## メイン機能の方向性（討論で決定）

「カードだけ投稿」では来店客の投稿モチベーションが弱い。
以下の組み合わせで**自然に投稿したくなる仕組み**にする。

### コア機能：自分の料理写真 + AIカード合成
- 来店客が撮った料理写真を選ぶ
- AIがその写真に合った感想コピーを生成
- おしゃれなフレーム＋コピーが写真に重なったカードが完成
- **「自分の体験を記録する」感覚**で使い、結果として店の宣伝になる

### サブ機能①：Googleマップ口コミ誘導
- カード生成後に「Googleマップでも感想を残しませんか？」と表示
- AIが生成したコピーをコピペで口コミに使える
- **店舗にとってはSNS投稿より直接的な価値**

### サブ機能②：特典クーポン表示（店舗がONにした場合のみ）
- カード生成 → 画面に「ドリンク1杯サービス」等のクーポン表示
- 投稿は任意、カードを作るだけで特典がもらえる

## セキュリティ設計

| 項目 | 方針 |
|------|------|
| 保存する | 店舗名、ロゴ、メニュー名、匿名集計データ |
| 保存しない | 来店客の名前、連絡先、SNSアカウント、位置情報 |
| カード生成 | セッション内で完結 → 閉じたら消える |
| SNS投稿 | 端末のOS共有機能を使うだけ（アプリはSNSに一切アクセスしない） |
| 写真 | アップロードされた写真はカード合成後にサーバーから削除 |
| Gemini API | 有料版を使用（ユーザー入力がモデル改善に使用されない） |
| APIキー | Cloud Functions経由（フロントに直接置かない） |

## カードデザインの差別化ポイント

| 要素 | 仕組み |
|------|--------|
| 店舗テーマ連動 | 居酒屋→和風、カフェ→ナチュラル等、自動でトーン調整 |
| 季節テンプレート | 桜・花火・紅葉・雪など季節ごとに自動切替 |
| AIコピー生成 | 「牛タン 柔らかい」→「とろける幸せ、ここにあり。」のように変換 |
| デザインシャッフル | 🔄ボタンで別デザインを何度でも試せる |
